## Workflow

The project follows this flow:

1. Raw data download from EU portal → `bases_datos_erasmus/` folder
2. Execution of `data_erasmus.ipynb` notebook
3. Generation of processed files (Parquet and CSV)
4. Loading into Power BI (Parquet or CSV)
5. Creation of visualizations and analysis

## Project Statistics

### Code

- **Main language**: Python 3.8+
- **Core libraries**: Pandas, NumPy
- **Notebook cells**: 33 (code + markdown)

### Data

- **Original dataset**: 5,955,075 records
- **Processed dataset**: 2,082,071 records (HE 2017-2024)
- **Reduction**: 65% (filtered by type and period)
- **Duplicates removed**: 9,165 (0.15%)

### Transformations

- **Original columns**: 19-20 (varies by year)
- **Final columns**: 21 (standardized)
- **New variables created**: 8 (isced_level, activity_group, etc.)
- **Normalized countries**: 255 → 33 unique ISO2 codes

## Maintenance

### Update with New Data (2025+)

When the European Commission publishes 2025 or later data:

1. **Download** new CSV to `bases_datos_erasmus/`
2. **Modify** range in notebook:
   ```python
   years = range(2014, 2026)  # Change from 2025 to 2026
   ```
3. **Execute** complete notebook
4. **Verify** column structure (there may be changes)
5. **Update** RENAME_MAP if there are new column names

### Add New Transformations

1. Add cells in the appropriate notebook section
2. Document in comments why the transformation is necessary
3. Update `PIPELINE.md` with the new phase
4. Update `DATA_DICTIONARY.md` if new variables are created
